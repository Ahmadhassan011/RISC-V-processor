User Guide

Introduction

Welcome to the RISC-V Visualizer, an interactive tool designed to help you understand the inner workings of a 5-stage RISC-V processor pipeline. This guide will walk you through the features of the visualizer and explain how to interpret the simulation data.

How to Use the Visualizer

  1. Enter Your Code:
    In the control panel on the main visualizer page, you can enter your RISC-V assembly instructions or hexadecimal machine code directly into the text area.
  2. Run the Simulation:
    Click the "Run Simulation" button to process your code. The visualizer will execute the instructions and display the state of the processor at each cycle.
  3. Navigate Through Cycles:
    Use the slider or the cycle navigation buttons to step through the simulation cycle by cycle. This allows you to observe how instructions move through the pipeline.

Understanding the Pipeline Stages

The visualizer displays the five stages of the RISC-V pipeline. Here's what each stage does:

  • IF (Instruction Fetch): Fetches the next instruction from memory.
  • ID (Instruction Decode): Decodes the instruction and reads the required values from the register file.
  • EX (Execute): Performs the calculation required by the instruction, such as an arithmetic operation.
  • MEM (Memory Access): Reads from or writes to memory if the instruction requires it.
  • WB (Write Back): Writes the result of the instruction back to the register file.

Interpreting the Data

As you navigate through the cycles, the visualizer will update to show you:

  • Register Values: The current values stored in each of the 32 general-purpose registers.
  • Pipeline Registers: The data being passed from one pipeline stage to the next.
  • Memory Contents: The current state of the data memory.
  • Control Signals: The signals generated by the control unit to manage the pipeline.
